## Codebase Patterns

### Ponder Indexer Patterns
- **Schema-first**: Always define Postgres schema in `ponder.schema.ts` BEFORE writing handlers
- **Table pluralization**: Ponder auto-pluralizes: `assets` table -> `assetss` in GraphQL (this is correct)
- **Column naming**: Use `snake_case` for all columns (e.g., `token_id`, `current_status`)
- **Upsert pattern**: Handlers use `context.db.<table>.upsert({ id, create: {...}, update: {...} })`
- **Index on filter fields**: Any field used in `where` clauses needs an index

### GraphQL Query Patterns
- Frontend queries are in `infrastructure/shared/graph-queries.ts` and `infrastructure/repositories/shared/graph-queries.ts`
- Ponder GraphQL filter syntax: `where: { field: value }` or `where: { field_in: [values] }`
- Response shape: `{ tableName: { items: [...], pageInfo?: {...} } }`

### File Locations
- Indexer schema: `indexer/ponder.schema.ts` (re-exports from `generated-schema.ts`)
- Indexer handlers: `indexer/src/handlers/`
- Indexer config: `indexer/ponder.config.ts`
- Contract addresses: `indexer/chain-constants.ts`
- Smoke tests: `indexer/test/smoke/`

### Event Table Naming Convention
- Raw event tables use pattern: `{eventName}_{signature4bytes}Events`
- Example: `unifiedOrderCreatedC8b6Events` for UnifiedOrderCreated event
- Aggregate tables use simple names: `assets`, `orders`, `journeys`

### Testing Commands
```bash
cd indexer && npx tsc --noEmit    # Type check indexer
cd indexer && npm run dev         # Start local indexer
cd indexer && npm test            # Run all tests
cd indexer && npm test -- --grep smoke  # Run smoke tests only
```
